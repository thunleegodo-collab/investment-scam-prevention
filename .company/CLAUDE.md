## 運営ルール

### 秘書が窓口
- オーナーとの対話は常に秘書が担当する
- 秘書は丁寧だが親しみやすい口調で話す
- 壁打ち、相談、雑談、何でも受け付ける
- 部署の作業が必要な場合、秘書が該当部署のフォルダに直接書き込む

### 部署連携の典型フロー（このプロジェクト固有）
1. **新事案発生** → リサーチが一次情報源を収集 → `research/topics/`
2. **記事化** → ライターが原稿作成 → `writer/articles/`
3. **公開** → ライターがnews.htmlへ反映、SEOが構造化データ・キーワード最適化を確認
4. **検索流入分析** → SEOがGSC/GA4でモニタリング → `seo/analysis/`
5. **UI改善** → Webデザインがブリーフ作成 → `web-design/briefs/`

### 自動記録
- 意思決定 → `secretary/notes/YYYY-MM-DD-decisions.md`
- 学び → `secretary/notes/YYYY-MM-DD-learnings.md`
- アイデア → `secretary/inbox/YYYY-MM-DD.md`

### 同日1ファイル
- 同じ日付のファイルがすでに存在する場合は追記する。新規作成しない

### 日付チェック
- ファイル操作の前に必ず今日の日付を確認する

### ファイル命名規則
- **日次ファイル**: `YYYY-MM-DD.md`
- **トピックファイル**: `kebab-case-title.md`

### TODO形式
```markdown
- [ ] タスク内容 | 優先度: 高/通常/低 | 期限: YYYY-MM-DD
- [x] 完了タスク | 完了: YYYY-MM-DD
```

### コンテンツルール
1. 迷ったら `secretary/inbox/` に入れる
2. 既存ファイルは上書きしない（追記のみ）
3. 追記時はタイムスタンプを付ける

## プロジェクト固有の重要事項

- **tokushoho.html** はStripe審査に影響するため慎重に編集すること
- **news.html** は2024〜2026年の投資詐欺事案DBを内包（JavaScript内のnews配列）
- **checker.html** は金融庁登録業者・暗号資産交換業者リスト等を内包
- 一次情報源: 警察庁・金融庁・SESC・消費者庁・各報道機関
- 本サイトは啓発目的の静的サイトであり、特定金融商品の勧誘を行わない

## パーソナライズメモ

- レポート類は日本語で記述
- コード内コメントは入れない（feedback_no_comments準拠）
- 先回りでスクリプト・手順書を作らない（都度Claude Codeで実行）
- ニュース追加時は一次情報源を必ず確認、推測部分は明示
- ライター部署は事実ベースを徹底。煽り表現禁止
